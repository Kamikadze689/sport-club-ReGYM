# Спортивный клуб ReGYM

ReGYM — это современная веб-платформа для управления спортивным клубом и фитнес-центром, разработанная на базе фреймворка Laravel. Система обеспечивает полноценное взаимодействие между клиентами и администрацией клуба, автоматизирует процессы покупки и проверки абонементов, ведение базы тренеров, гибкую настройку прайс-листов с динамическими характеристиками, а также включает в себя визуальный конструктор зонирования тренировочного пространства.

---

## Основной функционал системы

### 1. Личный кабинет клиента (Пользовательский интерфейс)
* Регистрация и авторизация: Безопасный вход, регистрация новых пользователей, восстановление пароля через почту и верификация Email.
* Покупка и продление абонементов: Выбор подходящих тарифов и проведение онлайн-оплаты через интегрированный платежный шлюз ЮKassa.
* Цифровой абонемент с QR-кодом: Автоматическая генерация уникального QR-кода при успешной оплате абонемента. Клиент может просмотреть, скачать или отправить на печать свой QR-код прямо из личного кабинета для прохождения через систему контроля доступа на ресепшене.
* Интерактивная карта клуба: Отображение актуальной планировки тренировочных зон зала в реальном времени с интерактивной разметкой.
* База тренеров: Информационный раздел со списком тренерского состава, их специализацией, видами спорта, стажем работы и личными цитатами.
* Форма обратной связи: Возможность быстро отправить заявку на обратный звонок или бронирование пробного посещения.

### 2. Панель администратора (Admin Dashboard)
* Сводная статистика: Информационная панель с ключевыми показателями (количество активных заявок от клиентов, общее число тренеров в базе, количество категорий услуг и тарифов).
* Управление тренерским составом: Полный цикл CRUD для администрирования карточек тренеров (добавление новых специалистов, редактирование опыта работы, загрузка фотографий в хранилище, указание спортивных направлений, сортировка вывода на главном сайте).
* Динамический конструктор тарифов:
  * Создание категорий услуг (например, Абонементы, Персональные тренировки, Секции).
  * Назначение произвольных свойств-характеристик для каждой категории (например, Время посещения, Количество занятий, Срок действия).
  * Создание тарифов (Price Items) с заполнением этих характеристик, которые автоматически сохраняются в базу данных в формате JSON.
  * Управление акционными предложениями (флаг скидки) и выделение популярных тарифов.
* Обработка клиентских заявок: Список всех входящих обращений с поддержкой пагинации, фильтрации по статусу обработки и возможностью отметить заявку как выполненную.
* Настройки клуба (Gym Info): Панель редактирования глобальных параметров организации (физический адрес, контактный телефон, email-адрес, часы работы в будние и выходные дни, рекламные тексты). Все данные кэшируются через глобальный сервис-провайдер и доступны на любой странице сайта.
* Конструктор планировки зала: Настройка интерактивной карты тренировочных зон (название зоны, цвет отображения, точные координаты сетки по осям X и Y, ширина и высота блоков на схеме).

### 3. Автоматизированные фоновые процессы
* Консольная команда `subscriptions:remind`: Фоновый процесс, сканирующий базу данных на наличие активных абонементов, срок действия которых завершается ровно через 7 дней.
* Автоматическая Email-рассылка: Отправка писем-напоминаний пользователям на электронную почту с помощью почтового класса `SubscriptionExpiringMail`.
* Защита от дублирования сообщений: Использование флага фиксации даты отправки уведомления в базе данных для предотвращения повторной отправки.

## Технологический стек

* Бэкенд-платформа: PHP 8.2+ / Laravel 11+
* База данных: MySQL
* Фронтенд-компоненты: Laravel Blade, Tailwind CSS, сборщик Vite
* Интеграция платежей: Пробный API-протокол ЮKassa
* Почтовый сервис: Laravel Mail (поддержка SMTP-протоколов и логирования в режиме отладки)
* Фоновые задачи: Планировщик задач Laravel Scheduler (Cron)

## Структура базы данных и ключевые модели

* User: Модель пользователей. Содержит флаг `is_admin` для разграничения прав доступа к панели управления и связь со списком абонементов.
* Subscription: Модель абонементов. Хранит дату начала, дату окончания, текущий статус (active/expired), ссылку на сгенерированный QR-код и дату отправки уведомления об истечении.
* Trainer: Модель тренеров. Хранит информацию о специалисте, включая опыт работы в годах, загруженную фотографию, массив специализаций и видов спорта (в формате JSON), а также поле `order` для кастомной сортировки.
* PriceCategory: Модель категорий прайс-листа.
* PriceCategoryProperty: Модель динамических свойств, привязанных к категориям прайса.
* PriceItem: Модель конкретных тарифных планов. Содержит цену, название, описание, флаги популярности и скидки, а также динамический JSON-объект `property_values` с заполненными свойствами.
* GymZone: Модель зон спортивного зала для отрисовки интерактивной карты (координаты сетки, размеры и стилистика).
* GymInfo: Модель хранения глобальных текстовых настроек клуба по принципу "ключ-значение".
* VisitorRequest: Модель регистрации клиентских заявок через внешние формы сайта.

## Инструкция по установке и локальному запуску

Для развертывания проекта на локальном компьютере выполните следующие шаги:

### Шаг 1: Клонирование репозитория
Склонируйте проект с удаленного репозитория и перейдите в его корневую директорию:
```bash
git clone [https://github.com/Kamikadze689/sport-club-ReGYM.git](https://github.com/Kamikadze689/sport-club-ReGYM)

Шаг 2: Установка зависимостей бэкенда

Установите все необходимые PHP-пакеты и библиотеки через Composer:
Bash

composer install

Шаг 3: Настройка окружения

Скопируйте файл конфигурации окружения по умолчанию:
Bash

cp .env.example .env

Откройте созданный файл .env в текстовом редакторе и укажите параметры подключения к базе данных, параметры почтового сервера для отправки уведомлений, а также ключи интеграции платежей ЮKassa:
Фрагмент кода

# Настройки ЮKassa для приема платежей
YOOKASSA_SHOP_ID=your_shop_id_here
YOOKASSA_SECRET_KEY=your_secret_key_here

# Настройки почты
MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=your_username
MAIL_PASSWORD=your_password
MAIL_FROM_ADDRESS="no-reply@regym.ru"
MAIL_FROM_NAME="Спортивный клуб ReGYM"

Шаг 4: Генерация ключа приложения

Сгенерируйте уникальный ключ безопасности Laravel:
Bash

php artisan key:generate

Шаг 5: Выполнение миграций и наполнение базы данных

Создайте необходимые таблицы в базе данных и заполните их демонстрационными системными данными:
Bash

php artisan migrate --seed

Шаг 6: Настройка символической ссылки для файлов

Создайте символическую ссылку, чтобы загружаемые изображения тренеров корректно отображались в публичной веб-директории:
Bash

php artisan storage:link

Шаг 7: Установка и сборка фронтенда

Установите JavaScript-зависимости проекта и запустите компиляцию стилей и скриптов через Vite:
Bash

npm install
npm run build

(Для работы в режиме активной разработки вместо npm run build используйте команду npm run dev).
Шаг 8: Запуск локального сервера разработки

Запустите встроенный веб-сервер Artisan:
Bash

php artisan serve

После запуска проект будет доступен в браузере по адресу: http://127.0.0.1:8000
